# 2. 핵심 사용자 시나리오

## 전체 흐름 요약

```
가입 → 연동 → 스티커판 생성 → 스티커 지급(반복) → 목표 달성/실패 → 보상/패널티 실행
```

---

## 시나리오: 커플 "민수 & 지은"

### STEP 1: 가입
- 민수가 앱을 설치하고 카카오 로그인으로 가입
- 닉네임: "민수", 프로필 사진 설정
- 온보딩 3장 (앱 소개 → 연동 방법 → 시작하기)

### STEP 2: 커플 연동
- 민수가 "파트너 초대" 버튼 클릭
- 6자리 초대코드 생성: `A3K7F2` (유효시간 24시간)
- 카카오톡으로 초대 링크 공유
- 지은이 링크를 눌러 앱 설치 후 가입
- 초대코드 자동 입력 → "민수님과 연결할까요?" → 수락
- 양쪽 모두에게 "연동 완료!" 알림

### STEP 3: 스티커판 생성
- 민수가 스티커판 생성 시작
- 설정 항목:
  - 스티커판 이름: "지은이 운동 챌린지"
  - 스티커를 모으는 사람: 지은
  - 스티커를 주는 사람: 민수
  - 목표 스티커 수: 20개
  - 기간: 30일
  - 성공 보상: "청담 오마카세" (텍스트로 입력)
  - 실패 패널티: ON → "민수에게 치킨 쏘기" (텍스트로 입력)
  - 스티커 이미지: 기본 별 스티커 선택 (또는 커스텀 업로드)
- "생성하기" 클릭 → 지은에게 푸시 알림: "새 스티커판이 생겼어요!"
- 지은이 앱에서 확인 후 "수락" → 스티커판 활성화

### STEP 4: 스티커 지급 (반복)
- 지은이 운동을 다녀옴
- 민수가 앱을 열고 "스티커 주기" 버튼 클릭
- 스티커에 한 줄 메모 추가 가능: "오늘 5km 뛰었대! 대단해 💪"
- 스티커 부여 → 지은에게 푸시 알림: "민수님이 스티커를 줬어요! (7/20)"
- 진행률 바가 35%로 업데이트

### STEP 5-A: 목표 달성 (성공)
- 28일째, 지은이 20번째 스티커를 받음
- 화면에 축하 애니메이션 + 효과음
- 스티커판 상태: `active` → `success`
- "보상 약속: 청담 오마카세" 카드가 표시됨
- 민수에게 알림: "지은님이 목표를 달성했어요! 보상을 실행해주세요"
- 민수가 "보상 완료" 버튼 클릭 → 보상 상태: `completed`
- (선택) 보상 인증 사진 업로드 가능

### STEP 5-B: 목표 실패 (기간 만료)
- 30일 경과, 스티커 15/20개
- 자정에 서버에서 자동 만료 처리
- 스티커판 상태: `active` → `failed`
- 양쪽에 알림: "스티커판이 만료되었어요"
- 패널티 ON이었으므로 → "지은님이 약속한 패널티: 민수에게 치킨 쏘기"
- 지은이 "패널티 완료" 버튼 클릭 → 패널티 상태: `completed`

### STEP 6: 히스토리
- "지난 기록" 탭에서 과거 스티커판 목록 확인
- 각 스티커판의 달성률, 기간, 보상/패널티 이행 여부 조회
- "올해 총 스티커 52개!" 같은 통계 표시

## 핵심 플로우 다이어그램

```
[가입/로그인]
     ↓
[관계 없음?] → [초대코드 생성/입력] → [관계 연동]
     ↓                                    ↓
[홈 대시보드] ←←←←←←←←←←←←←←←←←←←←←←←←←↓
     ↓
[스티커판 생성]
     ↓
[상대방 수락]
     ↓
[스티커 지급] ←→ 반복
     ↓
[목표 달성?]
   ├─ YES → [성공 보상 실행] → [히스토리 저장]
   └─ NO (기간 만료) → [실패 처리]
                          ├─ 패널티 ON → [패널티 실행] → [히스토리 저장]
                          └─ 패널티 OFF → [히스토리 저장]
```
