# 4. 화면 기획

## 화면 목록 및 네비게이션 구조

```
[온보딩] → [로그인] → [관계 연동] → [홈]
                                      ├── 스티커판 생성
                                      ├── 스티커판 상세
                                      │    └── 스티커 주기
                                      ├── 히스토리
                                      └── 설정
```

---

## 4-1. 온보딩 화면 (3장 스와이프)

| 항목 | 내용 |
|------|------|
| **목적** | 앱의 핵심 가치를 3초 안에 전달 |
| **UI 요소** | 일러스트 이미지 + 짧은 설명 텍스트 (3페이지), 페이지 인디케이터, "시작하기" 버튼 |
| **페이지 내용** | ① "서로에게 칭찬 스티커를 주세요" ② "목표를 달성하면 보상이 기다려요" ③ "지금 시작해볼까요?" |
| **주요 버튼** | "건너뛰기" (우상단), "시작하기" (마지막 페이지) |
| **사용자 행동** | 좌우 스와이프 또는 건너뛰기 → 로그인 화면으로 이동 |
| **필요한 데이터** | 없음 (정적 콘텐츠) |

---

## 4-2. 회원가입 / 로그인 화면

| 항목 | 내용 |
|------|------|
| **목적** | 최소한의 입력으로 가입 완료 |
| **UI 요소** | 앱 로고, 소셜 로그인 버튼 3개 (카카오/구글/애플), 이용약관·개인정보 동의 링크 |
| **주요 버튼** | "카카오로 시작하기" (노란색, 가장 큼), "Google로 시작하기", "Apple로 시작하기" |
| **사용자 행동** | 소셜 로그인 클릭 → OAuth 인증 → 첫 가입이면 닉네임 입력 → 관계 연동 화면으로 이동 |
| **필요한 데이터** | OAuth 토큰, 닉네임, 프로필 이미지 URL |

**참고**: 애플 로그인은 iOS 앱스토어 출시 시 필수 (Apple 정책). MVP에서는 카카오+구글만 지원하고, 스토어 출시 시 애플 추가.

---

## 4-3. 관계 연동 화면

| 항목 | 내용 |
|------|------|
| **목적** | 파트너와 1:1 연결 |
| **UI 요소** | 두 가지 탭: "초대하기" / "코드 입력하기" |
| **초대하기 탭** | 6자리 초대코드 표시, "코드 복사" 버튼, "카카오톡으로 공유" 버튼, 유효시간 카운트다운 (24h) |
| **코드 입력 탭** | 6자리 입력 필드, "연결하기" 버튼 |
| **주요 버튼** | "코드 복사", "카카오톡으로 공유", "연결하기", "나중에 하기" |
| **사용자 행동** | (A) 초대코드 생성 → 공유 → 대기 / (B) 받은 코드 입력 → 연결 확인 팝업 → 수락 |
| **필요한 데이터** | 초대코드, 상대방 닉네임·프로필 사진 (연결 확인 시 표시) |

**에러 케이스**: 만료된 코드, 잘못된 코드, 이미 연결된 사용자 → 각각 다른 에러 메시지 표시

## 4-4. 홈 대시보드

| 항목 | 내용 |
|------|------|
| **목적** | 현재 상태를 한눈에 파악, 주요 액션 진입점 |
| **UI 요소** | 상단: 파트너 프로필 (닉네임 + 사진), 관계 상태 배지 |
|  | 중앙: 진행 중인 스티커판 카드 (최대 3개 표시) - 각 카드에 이름, 진행률 바, 남은 일수 |
|  | 하단: "새 스티커판 만들기" 버튼 |
|  | (광고 위치): 스티커판 카드 아래에 배너 광고 1개 |
| **주요 버튼** | "새 스티커판 만들기" (FAB 또는 큰 버튼), 스티커판 카드 탭 → 상세로 이동 |
| **사용자 행동** | 앱 열기 → 현재 진행 상황 확인 → 스티커판 탭하여 상세 보기 또는 새로 만들기 |
| **필요한 데이터** | 파트너 정보, 활성 스티커판 목록 (이름, 현재/목표 수, 시작일, 종료일) |

**빈 상태**: 연결된 파트너 없으면 → "파트너를 초대해보세요" CTA. 스티커판 없으면 → "첫 스티커판을 만들어보세요" CTA.

## 4-5. 스티커판 생성 화면

| 항목 | 내용 |
|------|------|
| **목적** | 새로운 챌린지/목표 설정 |
| **UI 요소** | 폼 형태, 단계별 또는 한 페이지 스크롤 |

### 입력 필드 상세

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 스티커판 이름 | 텍스트 (30자) | 필수 | "지은이 운동 챌린지" |
| 스티커를 모으는 사람 | 선택 (나/상대방) | 필수 | 드롭다운 |
| 목표 스티커 수 | 숫자 (5~100) | 필수 | 슬라이더 + 직접입력 |
| 기간 | 날짜 선택 | 필수 | 시작일(오늘) + 종료일, 또는 "7일/14일/30일/직접입력" |
| 성공 보상 | 텍스트 (100자) | 필수 | "청담 오마카세" |
| 실패 패널티 | 토글 ON/OFF + 텍스트 | 선택 | OFF가 기본값 |
| 스티커 이미지 | 이미지 선택 | 필수 | 기본 스티커 6종에서 선택 (MVP), 커스텀 업로드 (2차) |

| **주요 버튼** | "미리보기", "생성하기", "뒤로 가기" |
| **사용자 행동** | 항목 입력 → 미리보기 확인 → 생성 → 상대방에게 수락 요청 알림 전송 |
| **필요한 데이터** | 파트너 닉네임 (드롭다운에 표시), 기본 스티커 이미지 목록 |

## 4-6. 진행 중인 목표 상세 화면

| 항목 | 내용 |
|------|------|
| **목적** | 스티커판의 현재 상태 확인 + 스티커 주기 |
| **UI 요소** | |

### 레이아웃

```
┌──────────────────────────────┐
│  "지은이 운동 챌린지"          │  ← 스티커판 이름
│  D-12  │ 18/20개             │  ← 남은 일수, 진행률
│                              │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐  │
│  │⭐│ │⭐│ │⭐│ │⭐│ │⭐│  │  ← 스티커 그리드
│  ├──┤ ├──┤ ├──┤ ├──┤ ├──┤  │    (채워진 것 + 빈 칸)
│  │⭐│ │⭐│ │⭐│ │⭐│ │⭐│  │
│  ├──┤ ├──┤ ├──┤ ├──┤ ├──┤  │
│  │⭐│ │⭐│ │⭐│ │⭐│ │⭐│  │
│  ├──┤ ├──┤ ├──┤ ├──┤ ├──┤  │
│  │⭐│ │⭐│ │⭐│ │  │ │  │  │
│  └──┘ └──┘ └──┘ └──┘ └──┘  │
│                              │
│  보상: 청담 오마카세           │
│  패널티: 치킨 쏘기            │
│                              │
│      [ 스티커 주기 🌟 ]       │  ← 주는 사람에게만 표시
│                              │
│  최근 기록                    │
│  • 4/5 "오늘도 5km!" - 민수   │
│  • 4/3 "필라테스 갔다옴" - 민수│
└──────────────────────────────┘
```

| **주요 버튼** | "스티커 주기" (주는 사람만), "스티커판 설정" (⚙️), "취소하기" |
| **사용자 행동** | 현황 확인 → 스티커 주기 → 메모 입력 → 확인 |
| **필요한 데이터** | 보드 정보, 스티커 목록 (시간순), 보상/패널티 텍스트 |

## 4-7. 스티커 주기 화면 (바텀시트)

| 항목 | 내용 |
|------|------|
| **목적** | 스티커를 부여하는 인터랙션 |
| **형태** | 바텀시트 (전체 화면 전환 없이 하단에서 올라옴) |
| **UI 요소** | 스티커 이미지 미리보기, 메모 입력 필드 (선택, 50자), "스티커 주기" 버튼 |
| **주요 버튼** | "스티커 주기" (확인), "취소" |
| **사용자 행동** | 메모 입력 (선택) → "스티커 주기" → 스티커 붙는 애니메이션 → 완료 |
| **필요한 데이터** | 현재 스티커 이미지, 현재 진행 수 |

**목표 달성 시**: 스티커 주기 후 목표 달성되면 → 축하 모달 자동 표시 ("축하해요! 🎉 목표를 달성했어요!")

## 4-8. 히스토리 화면

| 항목 | 내용 |
|------|------|
| **목적** | 과거 스티커판 기록 조회 |
| **UI 요소** | 카드 리스트 (최신순), 각 카드: 이름, 기간, 달성률, 상태 배지 (성공🟢/실패🔴/취소⚪) |
| **필터** | 전체 / 성공 / 실패 / 취소 |
| **카드 탭 시** | 상세 화면으로 이동 (읽기 전용) |
| **주요 버튼** | 필터 탭, 카드 탭 |
| **사용자 행동** | 과거 기록 탐색 → 특정 스티커판 상세 확인 |
| **필요한 데이터** | 비활성 스티커판 목록 (상태: success/failed/cancelled) |

## 4-9. 설정 화면

| 항목 | 내용 |
|------|------|
| **목적** | 계정 및 앱 설정 관리 |
| **UI 요소** | 섹션별 리스트 |

### 설정 항목

| 섹션 | 항목 |
|------|------|
| 프로필 | 닉네임 변경, 프로필 사진 변경 |
| 관계 | 파트너 정보, 관계 해제 |
| 알림 | 스티커 알림 ON/OFF, 만료 임박 알림 ON/OFF |
| 앱 정보 | 버전, 이용약관, 개인정보처리방침 |
| 계정 | 로그아웃, 회원 탈퇴 |

| **주요 버튼** | 각 항목 탭, "로그아웃", "관계 해제", "회원 탈퇴" |
| **사용자 행동** | 설정 변경 → 즉시 저장 |
| **필요한 데이터** | 유저 프로필, 파트너 정보, 알림 설정 상태 |

**위험 액션 처리**: "관계 해제"와 "회원 탈퇴"는 확인 모달 2단계 (1. "정말 해제할까요?" 2. "진행 중인 스티커판이 모두 종료됩니다. 계속할까요?")
