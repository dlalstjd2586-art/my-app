# 9. 광고 수익 모델 설계

## 9-1. 광고 유형별 분석

| 광고 유형 | UX 영향 | 수익성 | 적합도 | 권장 |
|----------|---------|--------|--------|------|
| **배너 광고** | 낮음 (항상 노출되지만 작음) | 낮음 (CPM $0.5~2) | 높음 | ✅ MVP |
| **전면 광고 (Interstitial)** | 높음 (화면 전체 차지) | 중간 (CPM $5~15) | 조건부 | ⚠️ 신중하게 |
| **리워드형 광고** | 낮음 (사용자 선택) | 높음 (CPM $10~30) | 높음 | ✅ 2차 |
| **네이티브 광고** | 매우 낮음 (콘텐츠와 자연스러운 통합) | 중간 | 낮음 (콘텐츠 부족) | ❌ 비적합 |

---

## 9-2. 광고 배치 전략

### 배너 광고 (MVP부터 적용)

| 화면 | 위치 | 이유 |
|------|------|------|
| 홈 대시보드 | 스티커판 카드 아래 | 가장 자주 보는 화면, 체류 시간 김 |
| 히스토리 목록 | 리스트 하단 | 과거 기록 조회 시 자연스러운 위치 |

**절대 넣지 말 곳**:
- 스티커 주기 화면 (핵심 인터랙션 방해)
- 스티커판 상세 화면 상단 (감성 파괴)
- 온보딩/로그인 화면 (첫인상 저해)

### 전면 광고 (조건부, 2차)

| 트리거 | 빈도 | 이유 |
|--------|------|------|
| 스티커판 생성 완료 후 | 생성 시 1회 | 자연스러운 전환 지점 |
| 히스토리 상세 닫을 때 | 3회 조회당 1회 | 과도하지 않게 제한 |

**절대 안 되는 타이밍**:
- 스티커 부여 직후 (축하 감성 깨짐)
- 목표 달성 축하 화면 (최악의 UX)
- 앱 열자마자 (이탈률 급증)

### 리워드형 광고 (2차, 게이미피케이션과 연결)

| 시나리오 | 보상 | 이유 |
|---------|------|------|
| "보너스 스티커 받기" | 광고 시청 → 스티커 1개 추가 | 사용자가 자발적으로 시청. 수집 동기와 연결 |
| "기간 연장" | 광고 시청 → 만료일 1일 연장 | 만료 임박 시 강한 동기 |

**주의**: 리워드형 광고로 스티커를 주면 "칭찬"의 의미가 퇴색될 수 있음.
→ 보너스 스티커는 별도 색상/디자인으로 구분하거나, "giver가 요청해야" 리워드 광고 시청 가능하게 제한.

## 9-3. 광고 정책 가이드라인

| 규칙 | 내용 |
|------|------|
| 하루 전면 광고 최대 노출 | 3회 |
| 배너 광고 새로고침 간격 | 30초 이상 (AdMob 정책 준수) |
| 리워드 광고 하루 시청 제한 | 5회 |
| 광고 제거 유료 옵션 | 현재 계획 없음 (무료 정책). 추후 프리미엄 모델 검토 가능 |
| 개인정보 광고 동의 | iOS ATT 팝업 필수 (IDFA 수집 동의) |

## 9-4. 예상 수익 분석 (현실적)

### 가정

| 지표 | 수치 | 근거 |
|------|------|------|
| 초기 DAU | 100~500명 | MVP 출시 후 3개월 |
| 배너 CPM | $1 (한국 평균) | 소규모 앱 기준 |
| 전면 CPM | $8 | |
| 리워드 CPM | $15 | |
| 1인당 일평균 세션 | 2회 | 커플앱 특성상 짧고 잦은 접속 |

### 월 수익 시뮬레이션

| DAU | 배너만 | 배너+전면 | 배너+전면+리워드 |
|-----|--------|----------|-----------------|
| 100명 | 약 $6/월 (≈8,000원) | 약 $10/월 | 약 $15/월 |
| 500명 | 약 $30/월 (≈40,000원) | 약 $50/월 | 약 $75/월 |
| 1,000명 | 약 $60/월 (≈80,000원) | 약 $100/월 | 약 $150/월 |
| 5,000명 | 약 $300/월 (≈400,000원) | 약 $500/월 | 약 $750/월 |
| 10,000명 | 약 $600/월 (≈800,000원) | 약 $1,000/월 | 약 $1,500/월 |

### 현실적 결론

> **DAU 1,000명 이하에서는 광고 수익이 서버 비용도 커버하기 어렵습니다.**
> - Supabase 무료 플랜으로 서버 비용을 최소화하는 것이 핵심
> - 광고 수익은 "있으면 좋은 것"이지 "수익 모델"이라 부르기엔 부족
> - DAU 5,000명 이상부터 의미 있는 수익 시작
> - DAU 50,000명 이상이면 월 수백만 원 가능

### 서버 비용 비교

| 서비스 | 무료 범위 | 유료 전환 시점 |
|--------|----------|--------------|
| Supabase | DB 500MB, Storage 1GB, 50K MAU | 충분 (초기) |
| Expo/EAS | 무료 빌드 30회/월 | 충분 (초기) |
| FCM (푸시) | 완전 무료 | - |
| AdMob | 무료 (수익 공유 모델) | - |

**결론: 초기에는 서버 비용 $0으로 운영 가능. 광고 수익은 DAU 성장에 비례하므로, 먼저 사용자를 모으는 데 집중.**

## 9-5. 광고 SDK 선택

| SDK | 장점 | 단점 | 권장 |
|-----|------|------|------|
| **Google AdMob** | 가장 보편적, 문서 풍부, Expo 지원 | 소규모 앱 CPM 낮음 | ✅ 1순위 |
