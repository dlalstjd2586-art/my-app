# 10. MVP 개발 우선순위

## 1차: 무조건 먼저 만들 기능 (Week 1~4)

> **목표: "두 사람이 스티커를 주고받고, 목표를 달성할 수 있다"**

| 순서 | 기능 | 예상 기간 | 이유 |
|------|------|----------|------|
| 1-1 | 프로젝트 초기 세팅 (Expo + Supabase) | 2일 | 모든 것의 기반 |
| 1-2 | 소셜 로그인 (카카오/구글) | 3일 | 사용자 식별 필수 |
| 1-3 | 프로필 설정 (닉네임) | 1일 | 가입 직후 필요 |
| 1-4 | 초대코드 기반 관계 연동 | 3일 | 2인 연동이 핵심 |
| 1-5 | 스티커판 생성 (이름, 목표 수, 기간, 보상 텍스트) | 3일 | 핵심 기능 |
| 1-6 | 스티커판 수락 | 1일 | 양쪽 동의 필요 |
| 1-7 | 스티커 부여 기능 | 2일 | 핵심 인터랙션 |
| 1-8 | 진행률 표시 (스티커 그리드 + 퍼센트) | 2일 | 동기부여의 핵심 |
| 1-9 | 목표 달성 자동 처리 | 1일 | 스티커 부여 시 체크 |
| 1-10 | 기간 만료 자동 처리 (Cron) | 2일 | 실패 판정 |
| 1-11 | 홈 대시보드 | 2일 | 앱의 메인 화면 |
| 1-12 | 기본 스티커 이미지 6종 | 1일 | 디자인 에셋 |
| 1-13 | 하단 네비게이션 (홈/기록/설정) | 1일 | 앱 구조 |

**1차 합계: 약 24일 (3~4주)**

---

## 2차: 출시 직후 붙일 기능 (Week 5~7)

> **목표: "사용 경험을 완성하고, 지속 사용을 유도한다"**

| 순서 | 기능 | 예상 기간 | 이유 |
|------|------|----------|------|
| 2-1 | 실패 패널티 설정 및 실행 | 2일 | 차별화 기능 |
| 2-2 | 푸시 알림 (스티커 받음, 달성, 만료 임박) | 3일 | 재방문율 핵심 |
| 2-3 | 히스토리 화면 | 2일 | 과거 기록 조회 |
| 2-4 | 보상/패널티 완료 확인 버튼 | 1일 | 이행 추적 |
| 2-5 | 스티커 메모 기능 | 1일 | 감성적 가치 |
| 2-6 | 배너 광고 (AdMob) | 2일 | 수익화 시작 |
| 2-7 | 설정 화면 (프로필 수정, 알림, 로그아웃) | 2일 | 기본 설정 |
| 2-8 | 관계 해제 기능 | 1일 | 필수 안전장치 |
| 2-9 | 보상에 외부 링크 저장 기능 | 1일 | 보상 유연성 |

**2차 합계: 약 15일 (2~3주)**

---

## 3차: 성장 후 붙일 기능 (Month 2~3+)

> **목표: "차별화하고, 수익을 키우고, 확장한다"**

| 순서 | 기능 | 예상 기간 | 이유 |
|------|------|----------|------|
| 3-1 | 커스텀 스티커 이미지 업로드 | 3일 | 개성 표현 |
| 3-2 | 보상 인증 사진 업로드 | 2일 | 신뢰성 강화 |
| 3-3 | 리워드형 광고 (보너스 스티커) | 3일 | 수익 증대 |
| 3-4 | 통계/리포트 (월간 스티커 수, 달성률) | 3일 | 장기 사용 동기 |
| 3-5 | 스티커판 취소 기능 | 1일 | 유연성 |
| 3-6 | 상대방 호칭 설정 | 1일 | 커스텀 경험 |
| 3-7 | 애플 로그인 (iOS 출시 시 필수) | 2일 | 앱스토어 정책 |
| 3-8 | SNS 공유 카드 이미지 생성 | 5일 | 바이럴 성장 |
| 3-9 | 홈 화면 위젯 | 5일 | 접근성 강화 |
| 3-10 | 다중 관계/그룹 지원 | 7일+ | 타겟 확장 |

---

## 출시 마일스톤

```
Week 0       : 프로젝트 세팅 시작
Week 4       : 1차 기능 완성 → 내부 테스트
Week 5       : 버그 수정 + 2차 기능 시작
Week 7       : 2차 기능 완성 → 클로즈드 베타 (지인 10~20명)
Week 8       : 베타 피드백 반영 + 버그 수정
Week 9       : 앱스토어/플레이스토어 심사 제출
Week 10      : 🚀 정식 출시
Week 11~14   : 3차 기능 순차 업데이트
```
