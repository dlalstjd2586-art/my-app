# 강의실 게이팅(B항목) 설치 가이드

강의실 페이지가 회원의 구독상태를 읽어 **구독중일 때만 비메오 영상을 노출**하도록 합니다.
방식: **비메오 도메인제한 + 구독상태 엔드포인트**.

## 잠금 2단계

1. **비메오 도메인 제한(1차·핵심)** — 영상 주소가 새도 *내 사이트 밖에선 재생 불가*
2. **엔드포인트 게이팅(2차)** — 강의실이 "이 회원 구독중?"을 물어 `시청가능`일 때만 영상 틀 노출

---

## STEP 1. 비메오 도메인 제한 설정

1. 비메오에서 강의 영상 열기 → **Settings(설정) → Privacy(개인정보)**
2. **"Where can this be embedded?"(임베드 허용 위치)** → **Specific domains(특정 도메인)** 선택
3. 도메인 추가:
   ```
   byediet.cafe24.com
   ```
   (실제 강의실이 열리는 도메인. 자체도메인을 쓰면 그 주소도 함께 추가)
4. 저장
5. (권장) **"Who can watch?"** 는 *Hide from Vimeo*(비메오에서 검색 안 됨)로

> 이렇게 하면 영상 ID가 노출돼도 등록된 도메인 밖에서는 재생되지 않습니다.

---

## STEP 2. 강의실 페이지에 코드 붙여넣기

1. 카페24 **쇼핑몰 관리 → 디자인(스마트디자인) → 강의실 페이지 HTML 편집**
2. 영상이 들어갈 자리에 `skin/lecture-gate.html` 내용을 그대로 붙여넣기
3. 붙여넣은 코드 위쪽 **설정 3곳**만 수정:

   | 항목 | 넣을 값 |
   |---|---|
   | `ENDPOINT_URL` | 우리 웹앱 주소(`.../exec`) — 이미 채워둠, 그대로 두면 됨 |
   | `VIMEO_ID` | 이 페이지에서 보여줄 비메오 영상 번호 |
   | `SUBSCRIBE_URL` | 강의구독 상품 구매 페이지 주소 |

4. 저장

> 강의가 여러 개면, 페이지마다 같은 코드를 넣고 `VIMEO_ID`만 그 강의 번호로 바꾸면 됩니다.

---

## STEP 3. 로그인 회원 아이디 연결 (한 번만 확인)

코드 안 `getMemberId()` 는 카페24 치환 변수 `{$member_id}` 로 로그인 회원 아이디를 읽습니다.
스킨/위치에 따라 이 변수가 안 바뀔 수 있어, **한 번만 확인**이 필요합니다.

**확인 방법**
1. 강의실 페이지에 로그인 상태로 접속
2. 브라우저에서 페이지 소스를 보거나, 콘솔에서 확인
3. `{$member_id}` 가 실제 아이디(예: `47744807@n`)로 바뀌어 있으면 → 끝, 그대로 사용
4. 만약 `{$member_id}` 글자 그대로 남아 있으면 → 스킨이 그 변수를 지원하지 않는 위치예요.
   알려주시면 그 스킨에 맞는 회원아이디 출력 방법으로 바꿔드립니다.

**테스트 팁**: 변수 확인 전이라도, 코드의 `getMemberId()` 위에
`window.MEMBER_ID = "47744807@n";` 한 줄을 임시로 넣으면 동작을 미리 시험할 수 있습니다.
(시험 후 반드시 지우세요.)

---

## 동작 확인

- **구독중(시청가능) 회원**으로 접속 → 영상이 보임 ✅
- **미구독 회원**(또는 비로그인)으로 접속 → 🔒 구독 안내 + 구매 버튼 ✅

문제가 있으면 어느 화면이 나오는지(영상/잠금/로딩 멈춤) 알려주세요.

---

## 보안 메모(이 방식의 범위)

- 영상 보호의 핵심은 **비메오 도메인 제한**입니다(STEP 1 필수).
- 엔드포인트는 회원ID로 구독상태(active)만 알려줍니다.
- 더 강한 본인확인이 필요하면, 회원별 토큰(이미 코드에 준비됨) 또는
  카페24 회원 추가정보 방식으로 강화할 수 있습니다. 필요 시 요청하세요.
